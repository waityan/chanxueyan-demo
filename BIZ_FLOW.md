# 业务流程设计文档

> 本文档定义 chanxueyan-demo 所有交互按钮背后的完整业务流程。
> 目标：每个按钮点击后都有完整的 front-end mock 流程（多步表单、状态变更、数据关联），而非单步 Toast。

---

## 一、报名参访流程

### 触发入口
路线详情页 → "立即报名" 按钮

### 流程步骤
```
[路线详情] → 点击"立即报名"
    → 报名表单（模态框）
        - 出行方式：自行前往 / 集合点集合（必选，单选）
        - 参与人数：1人 / 2人 / 3人 / 4人 / 5人（必选，默认1人）
        - 联系电话：（必填，预填 "138xxxx" 占位）
        - 备注留言：（可选）
    → 提交 → 校验（必填项）
        → 成功
            - 生成报名记录 → registrations[] 
            - 该路线卡片显示"已报名"标签
            - Toast "报名成功"
        → 失败 → Toast 提示必填项
```

### 数据模型
```js
// 新增到 MOCK_DATA
registrations: [
    {
        id: 1,
        activityId: 101,
        activityTitle: '大湾区黑科技探秘之旅',
        date: '07月12日',
        transport: '自行前往',     // 或 '集合点集合'
        participants: 2,
        phone: '13812345678',
        note: '',
        status: '已报名',
        createdAt: '2025-07-01T10:00:00'
    }
]
```

### 关联影响
- 路线卡片（列表和详情）显示 "已报名" 标签，再次点击变为灰色（"已报名，不可重复报名"）
- "我的" → "我的票券" → 跳转到报名列表页（显示所有 registration 记录）
- "我的" → "参访行程" → 同报名列表页

## 二、职位投递流程

### 触发入口
职位详情页 → "立即投递" 按钮

### 流程步骤
```
[职位详情] → 点击"立即投递"
    → 投递表单（模态框，两步骤）
        第一步：选择简历
            - 使用默认简历（MOCK_DATA.user 信息自动填充）
            - 或 新建简历 → 展开表单（姓名、学校、专业、电话、邮箱）
        第二步：补充信息
            - 自我评价：（可选文本框，placeholder "简单介绍你的优势..."）
    → 提交 → Toast "投递成功 🎉"
        - 生成投递记录 → applications[]
        - 该职位卡片显示"已投递"标签
```

### 数据模型
```js
// 新增到 MOCK_DATA
applications: [
    {
        id: 1,
        jobId: 201,
        jobTitle: '自动驾驶算法实习生',
        company: '小鹏汽车',
        resume: { name, school, major, phone, email },
        selfIntro: '',
        status: '已投递',
        createdAt: '2025-07-01T10:00:00'
    }
]

// 简历数据（用户可维护多份）
resumes: [
    {
        id: 'default',
        name: '陈小希',
        school: '华南理工大学',
        major: '计算机工程',
        phone: '13812345678',
        email: 'chenxiaoxi@example.com'
    }
]
```

### 关联影响
- 职位卡片显示 "已投递" 标签，再次点击不可重复投递（或提示"已投递"）
- "我的" → "投递记录" → 投递列表页（显示所有 application 记录）

## 三、课程加入流程

### 触发入口
课程详情页 → "查看课程" 按钮

### 流程步骤
```
[课程详情] → 点击"查看课程"
    → 免费课程：直接加入
        - Toast "已加入学习 🎉"
    → 付费课程：确认弹窗（当前仅 mock 数据都是免费或低价）
        - 确认 → Toast "购买成功"
    → 生成学习记录 → enrollments[]
    → "我的" → "已购课程" 可见
```

### 数据模型
```js
// 新增到 MOCK_DATA
enrollments: [
    {
        id: 1,
        courseId: 301,
        courseTitle: 'AI 时代下的产品思维升级',
        teacher: '腾讯资深导师',
        progress: 0,         // 学习进度 0-100
        enrolledAt: '2025-07-01T10:00:00'
    },
    {
        id: 2,
        courseId: 302,
        courseTitle: '从 0 到 1 掌握大厂面试技巧',
        teacher: '大疆 HR 专家',
        progress: 0,
        enrolledAt: '2025-07-01T10:00:00'
    }
]
```

### 关联影响
- "我的" → "已购课程" → 课程列表页（从 enrollments 读取）
- 课程卡片在列表页不重复显示"已加入"标签（简单处理）

## 四、发帖增强

### 触发入口
社区页 FAB 按钮

### 现有功能（保留并增强）
- 输入内容（已有）
- 选择话题标签（已有）
- 发布后插入 posts 列表首位（已有）

### 增强项
- 新增配图选择：点击图片按钮 → 弹出预设图库（从 demo-enhanced-assets 中选择 0-3 张）
- 发布后帖子作者显示 "我" 标识
- 自己的帖子在"我的" → "我的社区"中可查看
- 自己的帖子可删除（帖子详情页更多菜单 → 删除）

### 数据模型
```js
// posts 现有字段不变，增强配图选择
posts: [
    {
        ...现有字段,
        isMine: true   // 新增：标记是否自己发的
    }
]
```

### 关联影响
- "我的" → "我的社区" → 显示用户发的帖子列表（filter isMine === true）
- 发帖人自己的帖子在社区帖子列表中显示 "我" 标签
- 删除帖子后社区列表刷新

## 五、"我的"页面数据看板

### 整体结构
```
我的页面（增强版）
├── 个人信息卡片（已有，保持不变）
├── 数据概览（新增统计数字）
│   ├── 我的票券：N 张（已有，改为从 registrations 读取）
│   └── 收藏内容：N 项（已有，从 mock 收藏数据读取）
├── 功能区
│   ├── 参访行程 → 报名列表页（改为显示 registrations）
│   ├── 已购课程 → 课程列表页（从 enrollments 读取）
│   ├── 投递记录 → 投递列表页（从 applications 读取）
│   └── 我的社区 → 我的帖子列表（filter isMine）
└── 设置 → 设置页面（已有）
```

### 新增页面

#### 报名列表页 (`renderMyRegistrations`)
显示所有 registrations，每条显示：路线名称、日期、出行方式、人数、状态标签

#### 投递列表页 (`renderMyApplications`)
显示所有 applications，每条显示：职位名称、公司名称、投递时间、状态标签

#### 课程列表页（已有 `renderCourses`，改为从 enrollments 读取）
显示已加入的课程列表

#### 我的帖子页 (`renderMyPosts`)
显示用户发的帖子列表（isMine === true）

## 六、UI/UX 细节

### 状态标签颜色
| 状态 | 颜色 |
|---|---|
| 已报名 | bg-blue-50 text-blue-600 |
| 已投递 | bg-blue-50 text-blue-600 |
| 已参加 | bg-emerald-50 text-emerald-600 |
| 已取消 | bg-slate-50 text-slate-400 |

### 按钮状态
- 已报名的路线：按钮变灰，文字改为 "已报名"，不可点击
- 已投递的职位：按钮变灰，文字改为 "已投递"，不可点击
- 已加入的课程：按钮变灰，文字改为 "已加入"，不可点击

### 交互反馈
- 所有表单提交前做必填校验
- 校验失败 Toast 提示具体字段
- 提交成功 Toast 成功后自动关闭模态框
- 模态框点击遮罩层不关闭（需点击 X 或取消按钮）

## 七、实现要求

1. **纯前端 Mock**：所有数据存在 MOCK_DATA 新增数组中，刷新后重置
2. **不引入外部库**：继续使用 Tailwind CDN + FontAwesome + 原生 JS
3. **保持设计风格**：遵循 DESIGN_STYLE.md 和 DESIGN_SYSTEM.md 的规范
4. **代码结构**：新增函数放在现有新增函数区（`// ============ 新增交互功能 ============`）
5. **数据模型**：新增数组直接追加到 MOCK_DATA 对象中
6. **命名规范**：全中文标签，函数名 camelCase
7. **不要改动已有的 renderHome/renderCommunity/renderJobs 等主 Tab 函数**，只新增页面和修改按钮行为
